Lecture 5

CSS
- A set of presentation rules for styling the web
- Separation of structure from style
- A CSS rule has two main parts: Selector, declarations

Selector
- #id: used to specify a style for a single, unique element. The id selector uses the id attribute of the HTML element, and is defined with a "#".
- .class used to specify a style for a group of elements. The class selector uses the HTML class attribute, and is defined with a "."

Markdown
- Markdown is a lightweight markup language that you can use to add formatting elements to plaintext text documents.
- Created by John Gruber in 2004, Markdown is now one of the world’s most popular markup languages.

Lecture 6

What is Version Control?
That saving the thing again and again is the goal and where version control helps, providing you clarity as to when you did it, why you did it, and what the contents of the change were, open for review at any time in the future.

A version control system is a system that records changes to a set of one or more files so that earlier versions of any of these files can be restored at a later time.

Benefits of Version Control Systems
- Long-term change history
- Branching and merging
- Traceability

Lecture 7

Project Evaluation Criteria

1. Is it open source?
2. What is its license?
3. How active is this project?
4. How welcoming does the community seem?
5. How easy is it to find information about contributing to the project?
6. What programming languages are used in the project?
7. What is the development environment, and how hard is it to download and install it?
8. How hard is it to understand the project code? Is it a large code base?
9. Does the project depend on external additional software modules such as database or graphics libraries?
10. How much does one need to know about the domain of the application in order to understand the code?
11. How complicated is the code and how large is it?
12. How mature is the project? Is it very new, or has it been around for a long time? Does it have a stable release yet?
13. Is the project interesting and/or exciting to you?

GitHub enable to websites for you and your projects. Jekyll is a static site generator. Jekyll is a Ruby Gem that can be installed on most systems.

Basic Jekyll structrue
- _config.yml is where you manage the configuration for your project, like global variables, “collections”, or default names/paths. This where a lot of customization is done
- .gitignore is for files/folders you don’t want to save into version control (e.g., information you don’t want available publicly).
- Gemfile/Gemfile.lock is how you manage any extensions to Jekyll.

Lecture 14

Main Challenges of ML Development
Managing the published ML models is as important as developing them. ML Solution is facing challenge the balck box problem.

What is MLOps?
MLOps is an approach to managing machine learning projects. MLOps bridges the gap between data scientists and operation teams and helps to ensure that models are reliable and can be easily deployed.

MLFlow
MLFlow is an experiment tracker and generic model server. It can be self-hosted.
MLFlow Components
- MLflow Tracking : Record and query experiments: code, data, config, and results
- MLflow Projects : - Package data science code in a format to reproduce runs on any platform
- MLflow Models : Deploy machine learning models in diverse serving environments
- MLflow Registry : Store, annotate, discover, and manage models in a central repository
